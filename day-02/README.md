# Day 02 - Users and Groups in Linux

## Objective

What was the goal for today?
To understand how Linux manages users and controls who can access or make changes to the system.
---

## What I Learned

- Linux allows multiple users to use the system at the same time.
- Not everyone should have full access, so permissions are used to control what each user can do.
- A user is an account (for a person or system process).
- A group is a set of users that share the same permissions.
- The root user is the most powerful user and can do anything on the system.
- Using users and groups helps keep the system safe and organized
- I also tried to rehearse some of my learnings from Day1
  
---

## What I Built / Practiced

- Checked who I am logged in as using the 'whoami' command
- Viewed my user and group details 'id'
- Created a new user: sudo useradd <>
- Set a password for the new user: sudo passwd <>
- Switch to a user: su <>

---

## Challenges Faced

- It was a bit confusing remembering all the commands.
- I wasn’t sure when I needed to use sudo.
- Understanding how groups actually affect permissions took some time. 

---

## Key Takeaways

- Linux is designed for multiple users, so access must be controlled.
- Users and groups help manage who can do what.
- The root user has full control, so it should be used carefully.
- Practice is important to get comfortable with the commands.
