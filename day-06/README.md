# Day 06 - The Linux Filesystem: Everything Starts at /

## Objective
Understand how Linux organises every file and folder
from a single root point, and navigate it confidently.

---

## What I Learned
- Linux has one single starting point called / (root) — no drive letters like Windows
- /home is where my personal files live
- /bin is where everyday commands like ls and cp are stored as programs
- /etc holds configuration and settings files for the system
- /var/log holds log files — critical for monitoring data pipelines
- /tmp is for temporary files that get wiped on restart
- /proc contains live system information generated by the kernel
- Absolute paths start from / and work from anywhere
- Relative paths start from where I currently am

---

## What I Built / Practiced
- Ran ls / to see the full root filesystem
- Explored /bin and found commands I already know living there
- Read /etc/shells and /proc/cpuinfo using cat
- Practised navigating with absolute paths using cd /var/log and cd /etc
- Used pwd after each cd to confirm where I landed

---

## Challenges Faced
- Need to go over the concepts again so that they stick better

---

## Key Takeaways
- Everything in Linux is a file — even live system info in /proc
- /var/log will matter a lot when I start building pipelines
- Absolute path = full address from /. Relative path = directions from where I am
- /root and / are completely different things despite sounding similar
