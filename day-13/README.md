## Objective
Learn how to chain commands together and redirect output in Linux.

---

## What I Learned
- The pipe | sends output of one command into another
- > writes output to a file, overwriting existing content
- >> appends output to a file without overwriting
- < takes input from a file instead of the keyboard

---

## What I Built / Practiced
- Chained commands together using the pipe operator
- Redirected output into files using > and >>

## Key Takeaways
- Piping is one of the most powerful concepts in Linux
- > is destructive — it overwrites. >> is safe — it appends
- Commands can be chained endlessly with pipes
